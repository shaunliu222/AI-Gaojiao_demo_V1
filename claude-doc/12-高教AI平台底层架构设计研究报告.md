## 一、项目背景与需求概览

### 1.1 产品定位

构建"双轨驱动"的高校大模型原生基座，包含两大子平台：

| 子平台 | 核心价值 | 用户群体 |
|--------|---------|---------|
| **高教 AI 中台** | 模型/Skill/MCP/智能体的统一管理与分发 | 信息中心员工 |
| **助学助教平台** | 教学/科研/实训垂类 AI 应用 | 教师、学生 |

## 二、大模型核心技术选型

### 2.1 RAG（检索增强生成）

| 技术阶段 | 原理 | 高教场景应用 | 推荐度 |
|---------|------|------------|--------|
| **Advanced RAG** | Query 改写 + HyDE + 重排序 + 分块优化 | 课程知识库问答（教材/PPT/论文检索） | **P0 首选** |
| **图谱增强 RAG** | 知识图谱结构检索 + 挂载知识片段 + 向量混合 | 岗位能力推理、跨课程知识关联、学习路径 | **P1 二期** |
| **Agentic RAG** | Agent 驱动的自适应检索策略 | 学生自主学习（根据问题复杂度自动选策略） | P2 三期 |
| **Modular RAG** | 检索-过滤-重排-压缩流水线 | 大规模教材库精确检索 | P1 |

**推荐方案**：
- 基础 RAG：RagFlow（文档解析最强）或 LlamaIndex（灵活编排）
- Embedding：**BGE-M3**（BAAI，中英文最佳，支持混合检索）
- 重排序：**BGE-Reranker-v2**
- 向量数据库：**Milvus 2.4**（大规模）或 **Qdrant**（性能/易用平衡）

### 2.2 Agent / 智能体技术

| 技术 | 说明 | 适用场景 |
|------|------|---------|
| **ReAct** | 推理-行动交替循环 | 通用 Agent 执行 |
| **Function Calling** | 模型结构化调用外部函数 | 工具精确调度 |
| **Multi-Agent** | 多 Agent 角色协作 | 出题 + 审题 + 评分协同 |
| **Plan-and-Execute** | 先规划再执行 | 多步骤教学任务编排 |

**OpenClaw 已有能力**：Agent Runtime 完整支持 ReAct + Function Calling + SubAgent。多 Agent 可通过 ACP spawn 实现。通过 Plugin SDK 注册教育专用工具。

### 2.3 知识图谱

**构建方法**：五步渐进式图谱构建（非 GraphRAG 自动生成，强调人机协同 + 持续生长）

```
Step 1: 骨架定义
  专家定义顶层框架（按教学大纲：章→节→知识点）
  或用户上传关键核心知识资料，LLM 辅助提取框架结构
      │
      ▼
Step 2: 实体/关系抽取
  教材/试题 → MinerU 解析为 Markdown → LLM 自动抽取实体和关系
  实体类型：知识点、概念、公式、方法、案例
  关系类型：前置依赖、包含、关联、应用于
      │
      ▼
Step 3: 人工校验
  人工校验和补充，形成图谱骨架结构
  可视化审核界面（AntV G6），支持拖拽编辑节点和关系
      │
      ▼
Step 4: 持续挂载（核心机制）
  后续上传的知识/资料/学习内容 → MinerU 解析为 Markdown
  → LLM 基于已有图谱结构和实体，进行智能切片和信息提取
  → 自动匹配关联到图谱的对应节点上
  → 图谱持续生长，知识片段不断挂载积累
      │
      ▼
Step 5: 图谱检索问答
  用户提问 → 检索图谱结构定位相关知识节点
  → 沿关系边扩展邻域 → 获取挂载的 Markdown 知识片段
  → 知识片段 + 图谱上下文注入 LLM → 生成有据可查的回答
```

**技术要点**：
- 文档解析统一走 MinerU → Markdown，保证知识切片质量
- 图谱结构（节点+关系）存 Neo4j/NebulaGraph
- 知识片段（Markdown 文本）存向量库（Milvus）+ 对象存储（MinIO）
- 节点 → 片段的挂载关系通过图数据库边关联
- 检索时先走图谱定位，再取片段做 RAG，兼顾结构化推理和文本语义

**图数据库选型对比**：

| 数据库 | 分布式 | 中文社区 | 许可证 | 推荐场景 |
|--------|--------|---------|--------|---------|
| **Neo4j 5** | 企业版 | 最大 | GPLv3/商业 | 知识图谱首选，生态最完善 |
| **NebulaGraph** | 原生分布式 | 国内活跃 | Apache 2.0 | 大规模图谱，国产信创 |
| **TuGraph** | 分布式 | 国内 | Apache 2.0 | 蚂蚁开源，信创合规 |

**推荐**：中小规模用 Neo4j Community；有国产化要求选 NebulaGraph。

### 2.4 模型微调

| 方法 | 参数量 | 显存需求 | 适用场景 |
|------|--------|---------|---------|
| **LoRA** | 0.1-1% | 16-24GB | **主流推荐**，性价比最高 |
| **QLoRA** | 0.1-1% | 8-12GB | 高校 GPU 资源受限时首选 |
| **DPO** | 偏好对齐 | 中 | 教学风格对齐，替代 RLHF |

**推荐**：框架用 **LLaMA-Factory**（一站式）；基座选 GLM-4 / Qwen2.5 / DeepSeek-V3；评测用 **OpenCompass**。

**关键建议**：先用 RAG 验证效果（覆盖 80% 需求），不满意再微调。

### 2.5 多模态技术

| 能力 | 方案 | 高教应用 |
|------|------|---------|
| 文档解析 | **MinerU**（上海 AI Lab） | 教材 PDF/PPT/Word 转结构化文本 |

| 语音交互 | Whisper-large-v3 + TTS | 口语练习、听力测试 |

### 2.6 安全与对齐

| 层次 | 技术方案 | 实现方式 |
|------|---------|---------|
| 输入过滤 | 关键词词库 + 分类模型 | Java 拦截层前置过滤 |
| Prompt Injection | System Prompt 隔离 + 检测模型 | OpenClaw Hooks 系统 |
| 输出审核 | 内容安全分类器 | 后置审核 + 人工抽检 |
| Guardrails | NeMo Guardrails | 声明式安全规则引擎 |
| 审计追溯 | 全量对话记录 | OpenClaw Session Transcript |
| 舆情拦截 | 热词动态更新 + 实时拦截 | Redis 热词库 + 中间件 |

### 2.7 MCP（Model Context Protocol）

**与 OpenClaw 的关系**：OpenClaw 已原生支持 MCP（`src/gateway/mcp-http.ts` + `src/agents/mcp-*.ts`），支持 stdio/SSE/HTTP 三种传输。

**高教场景 MCP Server 规划**：

| MCP Server | 功能 | 实现方式 |
|-----------|------|---------|
| 知识库 MCP | 向量检索 + 知识图谱查询 | @modelcontextprotocol/sdk (TS) |
| 教务系统 MCP | 成绩查询、选课、课表 | 对接教务 API |
| 题库 MCP | 出题、组卷、评分 | 数据库 CRUD |
| 代码评测 MCP | 编程题自动评测 | Docker 沙箱执行 |
| 文献检索 MCP | 学术搜索、论文获取 | 对接学术搜索 API |

### 7.2 岗位能力图谱设计

**数据模型**：

```
岗位 (Job) ──需要──→ 能力 (Competency) ──前置──→ 能力
    │                     │
    │                     ├──对应──→ 课程 (Course)
    │                     │
    │                     └──评估──→ 知识点 (Knowledge)
    │
    └──属于──→ 行业 (Industry)
```

**核心流程**：
1. **岗位分析**：从 JD 文本用 NLP 实体抽取技能词 → 映射到技能标准体系（ESCO/O*NET）
2. **能力评估**：学生做测评 → 生成当前能力画像
3. **Gap 分析**：当前能力 vs 目标岗位要求 → 差值 = 学习缺口
4. **路径规划**：基于知识图谱拓扑排序 → 按前置依赖生成学习序列
5. **资源推荐**：匹配课程/教材/练习题

**参考标准**：ESCO（欧洲技能分类）、O*NET（美国职业网络）、Lightcast Open Skills

### 7.3 知识图谱在教育中的应用

**五步渐进式构建方法**（详见 2.3 节）：

1. **骨架定义**：专家按教学大纲定义顶层框架（章→节→知识点），或用户上传核心知识资料由 LLM 辅助提取
2. **实体/关系抽取**：教材/试题先经 MinerU 解析为 Markdown，再由 LLM 抽取实体和关系
3. **人工校验**：人工校验和补充，形成图谱骨架结构
4. **持续挂载**（核心机制）：后续上传的知识/资料/学习内容先解析为 Markdown，LLM 基于已有图谱结构和实体进行切片和信息提取，不断挂载关联在图谱节点上，图谱持续生长
5. **图谱检索问答**：用户提问时，通过检索图谱找到关联的知识节点，获取挂载的 Markdown 知识片段，注入 LLM 生成回答

**图谱检索问答流程**：
```
学生提问
    │
    ▼
图谱结构检索（Neo4j Cypher 查询 → 定位相关知识节点）
    │
    ├── 沿关系边扩展邻域（前置依赖、关联概念）
    │
    ▼
获取挂载的 Markdown 知识片段（从 Milvus 向量检索 + MinIO 原文）
    │
    ▼
知识片段 + 图谱上下文 → 注入 LLM Prompt
    │
    ▼
生成有据可查的回答（含知识点引用 + 图谱路径溯源）
```

**与岗位能力图谱的联动**：
- 岗位所需技能 → 映射到知识图谱中的知识点集合
- 学生当前能力评估 → 标记已掌握/未掌握的知识节点
- Gap = 未掌握但岗位要求的知识节点 → 沿前置依赖排序 → 生成学习路径
- 每个知识节点上挂载的 Markdown 资源 → 即为推荐的学习材料
