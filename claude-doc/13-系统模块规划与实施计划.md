# 高教 AI 教育平台 Demo -- 系统模块规划与实施计划

## Context

基于 `claude-doc/12-高教AI平台底层架构设计研究报告.md` 的架构设计，开发一个**全功能可运行的 Demo 原型**。目标是让信息中心、教师、学生能实际使用的高教 AI 教育平台，包含高教 AI 中台和助学助教平台两大子系统。

技术栈：React + Ant Design Pro（前端）、Spring Boot（业务后端）、OpenClaw（AI Gateway/Agent）。

## 三、各模块详细设计

### 【前端模块】

#### M1: IM 聊天界面（P0，核心入口）

- **功能**：
  - MainAgent 对话入口，支持文本/图片/文件发送
  - 流式输出（SSE/WebSocket），实时显示 Agent 回答
  - Agent 切换（从 MainAgent 路由到垂类 Agent）
  - 对话历史管理（会话列表、搜索、删除）
  - 知识引用展示（回答中引用知识来源、图谱节点）
  - 公共 Agent 列表 + 个人 Agent 列表
- **技术**：
  - WebSocket 直连 OpenClaw Gateway (:18789)
  - 聊天组件：@chatscope/chat-ui-kit-react 或自研
  - Markdown 渲染：react-markdown + rehype
- **关键页面**：`/chat`, `/chat/:sessionId`

#### M2: 管理后台布局（P0）

- **功能**：
  - Ant Design Pro 标准布局（侧边栏 + 顶栏 + 内容区）
  - 菜单权限动态加载（基于角色）
  - 多标签页切换
  - 全局通知和消息
  - 响应式布局
- **关键路由**：
  ```
  /dashboard          -- 总览面板
  /chat               -- IM 聊天
  /resources/*        -- 资源中心
  /agents/*           -- Agent/Skill 管理
  /knowledge/*        -- 知识图谱
  /competency/*       -- 能力图谱
  /users/*            -- 用户权限
  /security/*         -- 安全策略
  /models/*           -- 模型管理
  /channels/*         -- 渠道管理
  /settings/*         -- 系统设置
  ```

#### M3: 资源中心管理（P0）

- **功能**：
  - 三大板块管理：知识图谱资料、能力图谱资料、教学资源库
  - 文件上传（拖拽批量上传，支持 PDF/PPT/Word/Markdown/音视频/代码）
  - 统一入库流程展示：上传 → 解析状态 → 向量化状态 → 图谱挂载状态
  - 资源列表（搜索、筛选、分页、标签）
  - 公共/私有切换，权限设置（按学院/角色）
  - 资源预览（PDF/Markdown 在线预览）
- **关键页面**：`/resources/knowledge`, `/resources/competency`, `/resources/library`, `/resources/upload`

#### M4: Agent/Skill/MCP 管理（P0）

- **功能**：
  - Agent 列表（公共 + 私有）
  - Agent 创建向导：选择搭建方式（OpenClaw / 低代码开发平台 / 高代码 API）
  - Agent 配置编辑（System Prompt、工具、MCP、模型选择）
  - Agent 测试对话
  - Skill 列表、创建、编辑
  - MCP Server 接入管理
  - Agent/Skill 权限设置（公共/私有，授权范围）
  - Agent 发布到指定入口（门户、飞书等）
- **关键页面**：`/agents/list`, `/agents/create`, `/agents/:id/edit`, `/agents/:id/test`, `/skills/*`, `/mcp/*`

#### M5: 权限管理（P0）

- **功能**：
  - 用户管理（列表、搜索、导入、CRUD）
  - 角色管理（管理员、信息中心、教师、学生、自定义角色）
  - 学院/部门组织架构管理
  - 权限分配（资源/Agent/Skill/MCP/模型 × 角色/学院/个人）
  - 公共/私有资产管理（谁能发布为公共，谁能访问）
  - CAS/SAML 统一认证配置
- **关键页面**：`/users/list`, `/users/roles`, `/users/orgs`, `/users/permissions`

#### M6: 安全策略管理（P0）

- **功能**：
  - 关键词策略管理（添加/批量导入/启用/停用）
  - 安全策略规则配置（按学院/角色分配不同策略）
  - 拦截日志查看（被拦截的对话记录）
  - 模型安全策略（不同角色可用的模型范围）
  - 对话审计日志
- **关键页面**：`/security/keywords`, `/security/policies`, `/security/audit`

#### M7: 知识图谱可视化（P0）

- **功能**：
  - 图谱可视化展示（AntV G6 力导向图）
  - 图谱浏览：节点展开/折叠、搜索定位、层级导航
  - 图谱编辑：添加/编辑/删除节点和关系（人工校验）
  - 知识片段查看：点击节点查看挂载的 Markdown 知识
  - 图谱构建状态：上传资料 → 解析 → 抽取 → 挂载的进度
  - 公共图谱 vs 个人图谱切换
- **关键页面**：`/knowledge/graph`, `/knowledge/upload`, `/knowledge/nodes/:id`

#### M8: 能力图谱可视化（P0）

- **功能**：
  - 岗位→能力→课程→知识点 四层图谱展示
  - 学生能力画像：当前掌握 vs 目标要求（雷达图/差距图）
  - 学习路径生成：Gap 分析后展示推荐学习路径
  - 图谱编辑：岗位定义、能力定义、映射关系管理
  - 用户自建能力图谱
- **关键页面**：`/competency/graph`, `/competency/jobs`, `/competency/path`, `/competency/assessment`

#### M9: 模型管理面板（P0）

- **功能**：
  - 模型列表（已接入的标准/微调模型）
  - 模型配置（API Key、端点、参数）
  - 模型权限（哪些角色可用哪些模型）
  - 用量统计（调用次数、Token 消耗、成本）
  - 模型测试（Playground）
- **关键页面**：`/models/list`, `/models/:id/config`, `/models/usage`, `/models/playground`

### 【后端模块】

#### M10: 认证与权限模块（P0）

- **技术**：Spring Security + Sa-Token + Casdoor
- **数据库表**：
  - `sys_user` -- 用户表
  - `sys_role` -- 角色表
  - `sys_user_role` -- 用户角色关联
  - `sys_permission` -- 权限表
  - `sys_role_permission` -- 角色权限关联
  - `sys_org` -- 组织/学院表
  - `sys_user_org` -- 用户组织关联
- **API**：
  - `POST /api/auth/login` -- 登录（支持账号密码 + CAS 回调）
  - `POST /api/auth/logout` -- 登出
  - `GET /api/auth/userinfo` -- 当前用户信息
  - `CRUD /api/users/*` -- 用户管理
  - `CRUD /api/roles/*` -- 角色管理
  - `CRUD /api/orgs/*` -- 组织管理
  - `POST /api/permissions/assign` -- 权限分配
  - `GET /api/permissions/check` -- 权限校验

#### M11: 资源中心服务（P0）

- **技术**：MinIO 存储 + MinerU 解析 + Milvus 向量化 + Neo4j 挂载
- **数据库表**：
  - `res_file` -- 文件元数据（名称、类型、大小、存储路径、状态）
  - `res_category` -- 资源分类
  - `res_file_permission` -- 资源权限（公共/私有/授权范围）
  - `res_parse_task` -- 解析任务队列
- **API**：
  - `POST /api/resources/upload` -- 文件上传
  - `GET /api/resources/list` -- 资源列表（分页/筛选/权限过滤）
  - `GET /api/resources/:id` -- 资源详情
  - `DELETE /api/resources/:id` -- 删除资源
  - `PUT /api/resources/:id/permission` -- 设置权限
  - `GET /api/resources/:id/parse-status` -- 解析进度
  - `POST /api/resources/batch-upload` -- 批量上传
- **后台任务**：
  - 文件上传 → MinIO 存储
  - 异步解析：调用 MinerU API 转 Markdown
  - 向量化：调用 Embedding API → 写入 Milvus
  - 图谱挂载：调用知识图谱服务关联节点

#### M12: Agent/Skill 管理服务（P0）

- **数据库表**：
  - `agent_definition` -- Agent 定义（名称、类型、配置、状态）
  - `agent_permission` -- Agent 权限（公共/私有/授权）
  - `agent_publish` -- Agent 发布记录（发布到哪个入口）
  - `skill_definition` -- Skill 定义
  - `mcp_server` -- MCP Server 配置
- **API**：
  - `CRUD /api/agents/*` -- Agent CRUD
  - `POST /api/agents/:id/test` -- Agent 测试对话
  - `POST /api/agents/:id/publish` -- 发布 Agent
  - `CRUD /api/skills/*` -- Skill CRUD
  - `CRUD /api/mcp-servers/*` -- MCP Server CRUD
  - `PUT /api/agents/:id/permission` -- 设置权限
- **与 OpenClaw 交互**：
  - 通过 OpenClaw OpenAI-compatible API 创建/管理 Agent
  - 通过 OpenClaw Gateway API 管理 Skill/MCP 配置

#### M13: 知识图谱服务（P0）

- **技术**：Neo4j + LLM 实体抽取 + Milvus 向量检索
- **数据库（Neo4j）**：
  - 节点：`KnowledgeNode`（章/节/知识点/概念/公式）
  - 关系：`PREREQUISITE`, `CONTAINS`, `RELATES_TO`, `APPLIES_TO`
  - 属性：`name`, `type`, `description`, `level`, `owner_id`, `is_public`
- **数据库（PostgreSQL）**：
  - `kg_graph` -- 图谱元数据（名称、创建者、公共/私有）
  - `kg_build_task` -- 图谱构建任务
  - `kg_attachment` -- 节点挂载的知识片段索引
- **API**：
  - `POST /api/knowledge/graphs` -- 创建图谱
  - `GET /api/knowledge/graphs/:id/nodes` -- 获取节点列表
  - `POST /api/knowledge/graphs/:id/nodes` -- 添加节点
  - `PUT /api/knowledge/graphs/:id/nodes/:nodeId` -- 编辑节点
  - `POST /api/knowledge/graphs/:id/build` -- 触发自动构建（LLM 抽取）
  - `GET /api/knowledge/graphs/:id/build-status` -- 构建进度
  - `POST /api/knowledge/query` -- 知识检索（图谱 + 向量混合）
  - `POST /api/knowledge/upload-and-attach` -- 上传知识并挂载到图谱

#### M14: 能力图谱服务（P0）

- **数据库（Neo4j）**：
  - 节点：`Job`(岗位), `Competency`(能力), `Course`(课程)
  - 关系：`REQUIRES`, `MAPS_TO`, `PREREQUISITE`
- **数据库（PostgreSQL）**：
  - `comp_graph` -- 能力图谱元数据
  - `comp_assessment` -- 学生能力评估记录
  - `comp_learning_path` -- 学习路径记录
- **API**：
  - `CRUD /api/competency/graphs/*` -- 图谱 CRUD
  - `CRUD /api/competency/jobs/*` -- 岗位管理
  - `POST /api/competency/assess` -- 能力评估（学生做测评）
  - `POST /api/competency/gap-analysis` -- Gap 分析
  - `GET /api/competency/learning-path/:userId` -- 获取学习路径
  - `POST /api/competency/generate-path` -- AI 生成学习路径

#### M15: 安全策略服务（P0）

- **数据库表**：
  - `sec_keyword` -- 安全关键词（词、分类、状态）
  - `sec_policy` -- 安全策略（名称、规则 JSON、适用范围）
  - `sec_audit_log` -- 审计日志（用户、对话、命中规则、处理结果）
  - `sec_model_policy` -- 模型安全策略（角色→可用模型列表）
- **API**：
  - `CRUD /api/security/keywords/*` -- 关键词管理
  - `POST /api/security/keywords/batch-import` -- 批量导入
  - `CRUD /api/security/policies/*` -- 策略管理
  - `GET /api/security/audit-logs` -- 审计日志查询
  - `POST /api/security/check` -- 内容安全检查（被 AI 网关代理调用）

#### M16: AI 网关代理模块（P0）

- **职责**：Spring Boot 后端与 OpenClaw Gateway 之间的桥梁
- **功能**：
  - 代理转发：前端聊天请求 → 权限校验 → 安全检查 → OpenClaw API
  - 鉴权注入：在请求头中注入用户身份和权限信息
  - 输入过滤：调用安全策略服务检查用户输入
  - 输出审核：检查 AI 回复内容安全性
  - SSE 流式转发：保持流式输出不断开
- **API**：
  - `POST /api/ai/chat` -- 聊天（代理到 OpenClaw）
  - `POST /api/ai/chat/stream` -- 流式聊天（SSE）
  - `GET /api/ai/sessions` -- 会话列表
  - `DELETE /api/ai/sessions/:id` -- 删除会话

#### M17: 模型管理服务（P0）

- **数据库表**：
  - `model_config` -- 模型配置（名称、Provider、API Key、端点、参数）
  - `model_permission` -- 模型权限（角色→模型）
  - `model_usage_log` -- 用量日志（用户、模型、Token 数、成本）
- **API**：
  - `CRUD /api/models/*` -- 模型配置 CRUD
  - `PUT /api/models/:id/permission` -- 模型权限设置
  - `GET /api/models/usage` -- 用量统计
  - `POST /api/models/:id/test` -- 模型测试

#### M18: 渠道管理服务（P0）

- **数据库表**：
  - `channel_config` -- 渠道配置（类型、Token、状态）
  - `channel_agent_binding` -- 渠道与 Agent 绑定关系
- **API**：
  - `CRUD /api/channels/*` -- 渠道 CRUD
  - `POST /api/channels/:id/bind-agent` -- 绑定 Agent
  - `GET /api/channels/:id/status` -- 渠道状态检查

## 六、关键技术实现要点

### 6.1 前端与大模型直连（通过后端代理）

```
前端 (React)
    │  POST /api/ai/chat/stream (SSE)
    ▼
Spring Boot (M16 AI 网关代理)
    │  1. Sa-Token 鉴权
    │  2. 安全关键词检查 (M15)
    │  3. 模型权限检查 (M17)
    ▼
OpenClaw Gateway (:18789)
    │  OpenAI-compatible API
    │  /v1/chat/completions
    ▼
Agent Runtime → LLM Provider (Qwen/GLM/DeepSeek)
```

### 6.2 MainAgent 路由架构

```
用户在 IM 输入 → MainAgent 接收
    │
    ├─ 意图识别（LLM Function Calling）
    │
    ├─→ OpenClaw Agent（通过 Skill/MCP 执行）
    ├─→ 低代码 Agent（通过低代码开发平台 API 调用）
    └─→ 高代码 Agent（通过后端 REST API 调用）
```

### 6.3 知识图谱五步构建

```
上传资料 → MinIO 存储
         → MinerU REST API 解析为 Markdown
         → LLM 从 Markdown 抽取实体/关系
         → 人工通过前端校验（AntV G6 可视化编辑）
         → 后续资料：LLM 基于图谱结构做切片和挂载
         → Markdown 片段 → Milvus 向量化
         → 查询时：Neo4j 图谱定位 + Milvus 语义检索 → LLM 回答
```
