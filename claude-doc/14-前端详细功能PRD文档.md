### 1.2 设计风格

- **配色**：白色背景 + 浅灰卡片 + 深色文字，强调色黑灰渐变（`linear-gradient(135deg, #1a1a2e, #16213e)` 用于侧边栏/按钮高亮/顶栏等强调区域）
- **圆角**：卡片圆角 12px，按钮圆角 8px，输入框圆角 8px
- **阴影**：卡片 `box-shadow: 0 1px 4px rgba(0,0,0,0.08)`
- **字体**：PingFang SC / -apple-system / Segoe UI，正文 14px，标题 16-20px
- **图标**：Ant Design Icons + 自定义 SVG（模型 Provider Logo）
- **标签**：圆角小标签，红色 = 私有，蓝色 = 官方，绿色 = 启用中，灰色 = 已停用
- **动效**：过渡 200ms ease，卡片 hover 微上浮

### P01: 对话页 `/chat`

**定位**：基于 OpenClaw 搭建的智能体交互窗口，消息与各渠道 IM 中 OpenClaw Agent 保持同步。

**参考**：截图 Image 7（Main Agent 对话界面）

#### 技术对接

```
前端 Chat 页面
    │  POST /v1/chat/completions (SSE stream)
    │  Header: x-openclaw-agent-id: <agentId>
    │  Header: Authorization: Bearer <gateway-token>
    │  Body: { model: "openclaw/<agentId>", messages: [...], stream: true }
    ▼
OpenClaw Gateway (:18789)
    │  resolveGatewayRequestContext → agentId
    │  Agent Runtime → LLM Provider
    ▼
SSE 流式响应 → 前端逐字渲染
```

#### 布局

```
┌──────────┬──────────────┬──────────────────────────────────┐
│ 侧边栏   │  会话面板     │         聊天主区域                │
│ (全局)   │  280px       │                                   │
│          │              │  ┌─ Agent选择 ──────────────────┐ │
│          │ [+ 新建对话]  │  │ 🤖 Main Agent ▾              │ │
│          │              │  └──────────────────────────────┘ │
│          │ 历史对话 🔍   │  （无模型选择，模型由Agent配置决定） │
│          │              │                                   │
│          │ ○ 请介绍一下… │        [Agent 头像]               │
│          │ ○ 世界上有…   │        Main Agent                │
│          │ ○ 帮我分析…   │     与 AI 互动，探索无限创意      │
│          │              │                                   │
│          │              │  ┌──────────────────────────┐    │
│          │              │  │ 输入消息...               │    │
│          │              │  │ [+] [📎]            [↑]  │    │
│          │              │  └──────────────────────────┘    │
│          │              │                                   │
│          │              │  ● 已连接 OpenClaw Gateway        │
│          │              │                                   │
│          │              │       ✨ 猜你想问 ✨              │
│          │              │  ┌──────────┐ ┌──────────┐       │
│          │              │  │ 快捷问题1 │ │ 快捷问题2 │       │
│          │              │  └──────────┘ └──────────┘       │
└──────────┴──────────────┴──────────────────────────────────┘
```

#### 功能点

| # | 功能 | 描述 | 交互细节 |
|---|------|------|---------|
| 1 | Agent 选择器 | 顶栏唯一下拉，选择 OpenClaw Agent | 仅显示通过 OpenClaw 搭建的 Agent（agentType=openclaw），分组：公共 / 我的。选择后通过 `x-openclaw-agent-id` header 路由 |
| 2 | ~~模型选择器~~ | 已删除 | 模型由 Agent 配置中的 model.primary 决定，前端不暴露模型切换 |
| 3 | 新建对话 | 会话面板顶部按钮 `+ 新建对话` | 创建新会话，关联当前选中的 Agent ID |
| 4 | 历史对话列表 | 会话面板，按时间倒序排列 | 显示 Agent 名称 + 摘要 + 时间。右键菜单：置顶/重命名/删除 |
| 5 | 欢迎态 | 新对话时显示 Agent 头像 + 名称 + 描述 | 下方展示"猜你想问"快捷提问 |
| 6 | 消息输入框 | 圆角输入框 | 左侧：[+] 文件上传、[📎] 引用知识库；右侧：[↑] 发送 |
| 7 | 连接状态 | `● 已连接 OpenClaw Gateway` 或 `● 未连接` | 通过 healthz 端点检测 Gateway 可达性 |
| 8 | 消息气泡 | 用户消息右对齐深色，AI 消息左对齐浅色 | 支持 Markdown、代码高亮、LaTeX |
| 9 | 流式输出 | SSE stream=true，逐字渲染 | 通过 OpenClaw `/v1/chat/completions` 的 SSE 响应实现 |
| 10 | 知识引用 | AI 回复中引用的知识片段 | 可折叠卡片，来源文档名+页码 |
| 11 | 文件发送 | 拖拽或点击上传图片/PDF/文档 | 上传后作为 message content 的 image_url 或 file 附件 |
| 12 | 多渠道同步 | 本页对话与飞书/微信等渠道的同一 Agent 会话同步 | 基于 OpenClaw sessionKey 机制，同一用户+Agent 的会话数据共享 |

#### 状态

- **空态**：无历史对话时，会话面板显示"暂无对话记录"
- **加载态**：消息发送后，显示 Agent 头像 + 加载动画（三点闪烁）
- **错误态**：网络错误/模型不可用时，消息区域顶部红色提示条
- **无模型态**：输入框置灰 + placeholder "暂无可用模型,暂不可发送消息"

### P03: 我的智能体 `/agents/mine`

#### Main Agent（默认智能体）

"我的智能体"列表中始终存在一个**不可删除的 Main Agent**，它与"对话"页面的默认 Agent 完全一致：

- **来源**：基于 OpenClaw 配置中 `agents.list` 里 `default: true` 的 Agent
- **行为**：对应 OpenClaw 的默认 Agent（`resolveDefaultAgentId`），对话页未选择其他 Agent 时自动路由到此
- **特征**：
  - 卡片置顶，带 `默认` 金色标签，不可删除
  - 可编辑配置（模型、Skills、MCP 等），编辑后同步更新 OpenClaw 配置
  - 对话页顶栏的 "Main Agent" 选项即指向此 Agent

### P14: 渠道管理 `/system/channels`

**渠道的核心作用**：将 Agent 的能力暴露到外部消息平台（飞书/微信/Web门户等），让用户无需登录管理后台即可与 Agent 交互。

**不同类型 Agent 的渠道接入方式差异**：

| Agent 类型 | 渠道接入方式 | 配置要点 |
|-----------|-------------|---------|
| **OpenClaw Agent** | OpenClaw Channel 插件原生支持 | 配置 Channel Token（如飞书 App ID/Secret）→ 在 OpenClaw 配置中绑定 Agent ID → 消息自动路由到指定 Agent |
| **低代码开发平台 Agent** | 通过 Webhook/API 桥接 | 渠道消息 → 平台后端转发 → 低代码平台 API → 返回结果 → 后端推送到渠道。需配置低代码平台的 Webhook 回调地址 |
| **高代码(API) Agent** | 通过后端代理转发 | 渠道消息 → 平台后端 → 调用自定义 API → 返回结果 → 后端推送到渠道。需确保 API 端点可达 |
